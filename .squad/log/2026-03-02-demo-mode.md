# Session Log — 2026-03-02 Demo Mode Implementation

**Date:** 2026-03-02
**Goal:** Implement built-in demo mode for PII-free screenshots and walkthroughs

## Summary

Implemented a full demo mode for FlightDeck allowing the app to run with synthetic data and zero M365 dependency. The session spanned strategy analysis (Maverick), fixture + interceptor build (Viper), and HTML/CSS wiring (Goose).

**Key artifacts:**
- `src/demo/fixture.json` — static synthetic data covering persisted state and canned WorkIQ responses
- `src/renderer/demo.js` — interceptor that detects `?demo=1`, seeds localStorage, and monkey-patches `runWorkiqJson()` to return fixture responses
- `--demo` CLI flag in `src/main/index.js` appends `?demo=1` to the BrowserWindow URL

**Architecture insight:** Radar items are ephemeral (DEC-010, not persisted). Only intercepting `runWorkiqJson()` at the single chokepoint in `renderer/json-parser.js` can populate the Radar tab offline. This ruled out simpler approaches (blob scrubbing, static fake data).

**Files modified:** `src/demo/fixture.json` (new), `src/renderer/demo.js` (new), `src/renderer/app.js`, `src/main/index.js`, `src/index.html`, `src/styles/layout.css`, `package.json`

**Outcome:** Demo mode fully functional. No production code paths altered — interceptor activates only with `?demo=1`.
