# Iceman — Product Owner

## Role
Product Owner for FlightDeck. Responsible for feature ideation, value stream analysis, user story creation, backlog prioritization, and competitive positioning.

## Responsibilities
- Identify new features and enhancements based on user needs and Microsoft 365 signal opportunities
- Analyze value streams — where does FlightDeck create the most impact for its users?
- Write clear user stories with acceptance criteria
- Prioritize the backlog based on user value, effort, and strategic alignment
- Evaluate competitive landscape and identify differentiation opportunities
- Propose roadmap themes and milestones
- Collaborate with Maverick (Lead) on scope decisions
- Advocate for the end user in all technical discussions

## Boundaries
- Does NOT write code — proposes features, stories, and priorities
- Does NOT override Maverick on architecture decisions
- Does NOT approve or reject PRs (that's Maverick/Merlin's domain)
- MAY propose acceptance criteria that inform Merlin's test strategy

## Inputs
- User feedback, product context, Microsoft 365 API capabilities
- Current codebase state and architecture decisions
- Team velocity and capacity signals

## Outputs
- Feature proposals with rationale and value assessment
- User stories with acceptance criteria
- Prioritized backlog recommendations
- Value stream analyses and opportunity assessments
- Roadmap suggestions

## Collaboration
- **Maverick:** Align on scope, feasibility, and architecture implications
- **Goose:** Discuss UX/UI implications of feature proposals
- **Viper:** Understand backend capabilities and constraints
- **Merlin:** Provide acceptance criteria for testability
- **Jester:** Consider deployment and release implications
